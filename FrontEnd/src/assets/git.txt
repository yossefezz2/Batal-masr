ازاي تحمل البروجيكت عندك


 هتدخل على المكان اللي عايز فولدر البروجيكت ينزل  فيه هتدوس كليك يمين هيكتبلك (git bash here) >دوس عليها


هيفتحلك ترمينال هتعمل فيها paste للكوماند دا  (git clone )

هيقعد يحمل شوية اول ما يخلص هتكتب (cd bus-booking-app)

مفروض ساعتها يكتبلك بالازرق بين قوسين كلمة (master)

ف هتقوم كاتب (npm i) و تسيبه يحمل حبة

اول ما يخلص و دا المهم هتكتب (git checkout -b اسم البرانش بتاعك ) هنا بتكتب اي اسم للبرانش انتا عايزه بس لازم يكون اسم هتفتكره مثلا البرانش بتاعي اسمه frontend#0

الامر دا وظيفته انه بيعمل برانش جديد ليك انتا بتسميه اللي انتا عايزه بتشتغل فيه لوحدك عشان محدش يدخل يغير في الشغل بتاعك و مينفعش نهائي اي حد يشتغل ع الماستر 

كدا انتا تقدر تبدأ شغل و لو كتبت في الترمينال npm start 
هتشوف ال app وهو شغال في ال browser

________________________________________

في حاجة مهمة كل مرة تيجي تشتغل لازم تعملها و هي انك تحمل اي تحديثات في الكود اي حد م التيم عملها ف هتكتب كدا بالترتيب 
1 - git checkout master
2-git pull origin master
3- git checkout اسم البرانش الخاص بيك
4- git pull origin master

كدا انتا جاهز تشتغل

_________________________________________

طيب انا خلصت شغل و ال app شغال والدنيا تمام وعايز ارفع شغلي على github

جوا vs code هتلاقي تحت السيرش علامة اسمه source control

هتدوس عليها تلاقي مربع تكتب فيه باختصار شديد انتا اشتغلت على ايه النهاردة و تدوس okay  على اي حاجة تجيلك

بعد كدا تفتح الترمينال و تكتب كدا

git push origin أسم البرانش بتاعك

هيحمل والدنيا تمام ف هتفتح لينك البروجيكت بقا  (https://github.com/devMenna/bus-booking-app.git)

هتلاقي مربع اخضر ظهر مكتوب عليه compare and pull request

بتدوس عليه بيحمل شوية بعدين يعملك علامة صح ان كل حاجة تمام و بتبقا عملت pull request 
بس كدا مفروض ان حد بيتأكد ان كل حاجة تمام و محدش اشتغل في مكان غلط و يعمل merge و دي مش حوار انا هعملها
